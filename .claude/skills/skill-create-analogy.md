# Skill: Create Analogies

## Core rule

Every analogy must come from everyday life, never from IT or programming. The user should understand the analogy without knowing any code.

## How to build an analogy

1. **Identify the core mechanic** of the technical concept (e.g., "transforms input to output", "stores state", "routes decisions").
2. **Find a real-world process** with the same mechanic (cooking, postal service, school, sports, shopping).
3. **Map the components** — each technical part should have a clear real-world counterpart.
4. **State the limits** — briefly say where the analogy breaks down so it doesn't mislead.

## Analogy bank (examples)

| Technical concept | Analogy |
|---|---|
| Hooks (React/lifecycle) | Phone reminders — they fire at specific moments to remind you to do something |
| Agents (AI) | A manager who delegates tasks to specialists and combines their reports |
| Tokens (LLM) | SMS character limits — the model reads text in chunks, not as whole words |
| Embeddings | GPS coordinates for words — similar meanings are close on the map |
| Loss function | A teacher's red pen — it measures how far off your answer is from correct |
| Gradient descent | Walking downhill in fog — you feel the slope and step toward the lowest point |
| Attention mechanism | A spotlight on stage — it focuses on the most important actors in each scene |
| Fine-tuning | Teaching a chef a new recipe — they already know how to cook, you just guide their existing skills |
| RAG | An open-book exam — the model can look up references before answering |
| Vector database | A library organized by topic similarity, not alphabetically |
| Batch processing | Washing clothes — you wait until you have a full load, not one sock at a time |
| API | A restaurant waiter — takes your order (request), brings it to the kitchen (server), returns with food (response) |

## Quality checklist

- [ ] Analogy uses zero technical terms
- [ ] A 12-year-old could understand it
- [ ] It maps at least 2 components of the concept
- [ ] Limitations are stated ("unlike X, in reality Y...")
- [ ] It connects to something the user encounters in daily life
